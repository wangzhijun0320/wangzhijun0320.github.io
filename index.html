java bean

javabean是一个遵循特定写法的java类：通常具备以下特点
1：这个Java类必须具有一个无参的构造函数
2：属性必须私有化
3：私有化的属性必须通过public类型方法暴露给其他程序，并且方法的命名也必须遵守一定的规则。

JavaBean的属性

　　JavaBean的属性可以是任意类型，并且一个JavaBean可以有多个属性。每个属性通常都需要具有相应的setter、 getter方法，setter方法称为属性修改器，getter方法称为属性访问器。
　　属性修改器必须以小写的set前缀开始，后跟属性名，且属性名的第一个字母要改为大写，例如，name属性的修改器名称为setName，password属性的修改器名称为setPassword。
　　属性访问器通常以小写的get前缀开始，后跟属性名，且属性名的第一个字母也要改为大写，例如，name属性的访问器名称为getName，password属性的访问器名称为getPassword。
　　一个JavaBean的某个属性也可以只有set方法或get方法，这样的属性通常也称之为只写、只读属性。

面向接口编程

首先面向接口编程和面向对象编程并不是平级的，它并不是比面向对象编程更先进的一种独立的编程思想，而是附属于面向对象思想体系，属于其一部分。或者说，它是面向对象编程体系中的思想精髓之一。
在系统分析和架构中，分清层次和依赖关系，每个层次不是直接向其上层提供服务（即不是直接实例化在上层中），而是通过定义一组接口，仅向上层暴露其接口功能，上层对于下层仅仅是接口依赖，而不依赖具体类。
好处：
首先对系统灵活性大有好处。当下层需要改变时，只要接口及接口功能不变，则上层不用做任何修改。甚至可以在不改动上层代码时将下层整个替换掉
接口体现的是一种规范和实现分离的设计哲学，充分利用接口可以极好地降低程序各模块之间的耦合，从而提高系统的可扩展性和可维护性。基于这种原则，通常推荐“面向接口”编程，而不是面向实现类编程，希望通过面向接口编程来降低程序的耦合。
降低程序耦合度，提高系统的可扩展性和维护性。

Chapter6 继承和多态

6.1何为继承

6.1.1继承共同行为

新的关键字extends。

6.1.2多态与is-a

在java中，子类只能继承一个父类，继承除了可以避免类间重复的行为定义外，还有个重要的关系就是子类和父类间还有is-a的关系。
父类不能是一种子类，但是可以强制要求编译程序不要罗嗦。
利用多态思想撰写的函数，函数的参数为父类类型的对象，以抽象方法讲解多态就是使用单一接口操作多种类型的对象。

6.1.3重新定义行为

在继承父类之后，定义与父类中相同的方法部署，但执行内容不同，这称为重新定义(Override)。
在JDK5之后支持标注，其中一个内建的标准标注就是@Override。如果在子类中某个方法前标注@Override，表示要求编译程序检查，该方法是不是真的重新定义了父类中的某个方法，如果不是的话，就会引发编译错误。

6.1.4抽象方法，抽象类

如果某方法区块中没有任何程序代码操作，可以使用 abstract表示该方法为抽象方法，该方法可以不用撰写{}区块，直接用“；”结束即可。
类中如果有抽象方法，表明这个类定义不完整，就不能生成实例，内含 抽象方法的类也必须定义为abstract。
子类如果继承抽象类，有两种方法。一种做法是继续表示该方法为abstract，第二种做法是操作抽象方法。如果都没有实施，那么将编译错误。

##6.2继承语法细节

6.2.1protected 成员

父类中的成员被声明为private，如果子类中的方法想要直接存取父类中的方法，则可以把父类中的方法定义为protected。
被定义为protected的成员。相同包中的类可以直接存取，不同包中的类可以在继承后的子类直接存取。

6.2.2重新定义的细节

如果想取得父类中的方法定义，可以在调用方法前，加上super关键字。
重新定义方法要注意，对于父类中的方法权限，只能扩大但不能缩小。若原来成员为public，子类中重新定义是不可谓private或者protected。

6.2.3再看构造函数

如果类有继承关系，在创建子类实例后，会先进性父类定义的初始流程，在进行子类定义的初始流程。
如果想执行父类中的构造函数，可以使用super()指定。

6.2.4再看final关键字

如果对象数据成员被声明为final，但是没有明确使用=指定值，那表示延迟对象成员的指定，在构造函数执行流程中，一定要有对该数据成员的指定值的动作，否则编译错误。

class前面也可以加上final关键字，如果class前面加上final关键字定义，表示这个类是最后一个了，不会再有子类继承这个类。

定义方法是，也可以限定该方法为final，这表示是最后一次定义方法了，也就是自雷不可以重新定义final方法。

6.2.5java.lang.Object

在Java中，子类只能继承一个父类，如果定义类是没有使用extends关键字指定继承任何类，那么一定是继承java.lang.Object。

如果有个需求是使用数组收集各种对象，那么声明为Object[]。

Chapter5对象封装

5.1何为封装

5.1.1封装对象初始流程

如果想存取对象的数据成员，可以通过“.”运算符加上数据成员名称。（在程序中出现重复的流程，往往意味着有改进的空间）
在对象数据成员前面加上this关键字，表明将参数指定给这个对象（this）的数据成员。

5.1.2封装对象操作流程

在类中定义的方法，如果不用返回值，方法名前面可以声明为void。
前面的重复流程，现在都封装到方法中，这么做的好处事使用时简单。

5.1.3封装对象内部数据

如果不想让用户看到的数据成员，可以用private声明，并提供取值方法。封装的目的主要是隐蔽对象细节，将对象当作黑箱进行操作。

5.2类语法细节

5.2.1public 权限修饰

如果想要其他包的类的类程序代码中存取某包的类或对象成员，则该类或对象成员必须是公开成员，在Java中要使用public加以声明。
可以把类声明为public，这表明它是一个公开类，可以在其他包的类中使用。
可以在构造函数上声明为public，这表示其他包中的类可以直接调用这个方法。
可以在方法上声明public，这表示其他包中的方法中可以直接调用这个方法。

5.2.2关于构造函数

创建对象时，数据成员就会初始化。
只有编译程序自动加入的构造函数名，才称为默认的构造函数，如果自行撰写无参数，没有内容的构造函数，就不称为默认的构造函数了。

5.2.3构造函数与方法重载

可以定义多个构造函数，只要参数类型或个数不同，这称为重载构造函数。定义方法时也可以进行重载，可以类似功能的方法提供同一名称，但是根据参数类型和个数的不同调用对象的方法。
返回值类型不可作为方法重载依据，编译程序时会出错。

###5.2.4使用this
除了被声明为static的地方外，this关键字可以出现在类中的任何地方。在Java中，this()代表了调用另一个已经定义的构造方法。
在创建对象之后，调用构造函数之前，若有想执行的流程，可以使用{}定义。
final关键字，如果局部变量声明了final，表示设置后就不能再变动，但是如果没有明确使用=指定值，那么表示延迟对象成员值得指定，在构造函数执行流程中，一定要对该数据成员指定值得动作，否则将编译错误。

5.2.5 static类成员

被声明为static的成员，是将类名称作为命名空间。也就是说，可以用类名.成员来获取成语。所以根据习惯，首字母大写就是类，类名称加上“.”运算符直接调用的就是static成员。
由于static成员属于类，而非个别对象，所以在static成员中使用this会是一种语义上的错误。
在static方法或者区域块中，可以使用static数据成员或方法成员。
如果有些动作想在位码加载后执行，则可以定义static区块。

5.2.6不定长度自变量

声明参数列是要在类型关键字后加上…。使用不定长度自变量时，方法上声明的不定长度参数必须是参数列的最后一个。
使用两个以上不定长度自变量也是不合法的。

5.2.7内部类

可以在类中定义类，这称为内部类。要使用内部类时，必须先建立外部类的实例。

5.2.8传值调用

Java当中只有传值调用。

怎么解决这种问题，可以使用java.util.BigDecimal。这个类还提供了plus()，substract(),multiply(),divids()等方法。

4.1.3 对象指定与相等性

Java并非完全的面向对象的程序语言，在Java中有两大类型系统，基本类型和类类型。
如果在操作对象，=是用在指定参考名称参考某个对象，而==是用在比较两个参考名称是否参考用一个对象。通俗的说就是，=是用在将某个名牌绑定到某个对象，而==是用在比较两个名牌是否绑定到了同一个对象。【.equals()可以自行定义如何比较两个对象的内含值】

4.2基本类型打包器

4.2.1打包基本类型

Long、Integer、Double、Float、Boolean等类都是所谓的打包器，正如此名称所示，这些类主要目的就是提供对象实例座位“壳”，将基本类型打包在对象之中，这样就可以操作这个对象。
如果要使用Integer打包int类型数据，方法之一就是使用new创建Integer实例时，传入int类型数据。

4.2.2自动装箱、拆箱

除了使用new关键字创建基本类型打包器外，从j2SE5.0之后提供了自动装箱和自动拆箱功能，可以这样打包基本类型：Integer num=10;//自动装箱 int foo=num;//自动拆箱。

4.2.3 自动装箱，自动拆箱的内幕

自动装箱和自动拆箱实际上是编译程序蜜糖，Integer num=100;编译程序会自动将程序代码展开为：Integer localInteger=Integer.valueOf(100);valueof()的操作数在-128~127中间。所以当Integer i1=200;Integer i2=200;时，i1和i2不会参考同一个实例，所以用==比较就会显示false。用.equals()比较内容是否相等。

4.3数组对象

4.3.1数组基础

数组在Java中就是对象。

4.3.2操作数组对象

int[] scores=new int[10];在Java中看到new，一定是建立对象，这个语法代表了数组就是对象。每个数据类型的数组元素都有初始值，如果默认的初始值不符合你的要求，则可以通过java.util.Arrays的fill()方法来设定数组的元素值。
如果想在new数组中一并指定初始值，则可以不必指定数组长度，int[] scores=new int[]{45,234,24,34,32};
数组既然是对象，对象时根据类而建立的实例，代表建立数组对象定义在哪儿？ 答案是由JVM动态产生。
类类型建立的数组：Integer scores=new Integer[3];那么这个程序片段建立了几个Integer对象呢？是0个。因为如果是类类型，这个片段的写法建立的数组，每个索引都是参考至null。

4.3.3数组复制

如果想进行数组复制，基本做法就是另行建立新数组，使用System.arraycopy()方法，有五个参数，分别是来源数组、来源起始索引、目标数组、目标起始索引、复制长度。还可以使用Arrays.copyOf()方法，这个方法有两个参数，分别是目标对象，目标对象长度。

4.4字符串对象

4.4.1字符串基础

如果有一个char[]数组，也可以使用new来创建String实例，例如：
char[] cs={‘h’,’e’,’l’,’l’,’o’};
String name=new String(cs);
如果有必要，也可以使用String的toCharArray()方法，将字符串以char[]数组返回。
char[] cs2=name.toCharArray();
如果要将字符串转换为整数，浮点数等基本类型，可以使用打包类提供的剖析方法。

4.4.2字符串特性

以“”包含的字符串，只要内容相同，无论在程序代码中出现几次，JVM都只会建立一个String实例，并在字符串池中维护。

4.4.3字符串编码

在windows默认的纯文本编辑器，应该是GB2312编码。在eclipse中建立.java原始码，在eclipse运行在简体中文windows中，默认应该是GB2312编码。然而写下的字符串在JVM中会是Unicode。
