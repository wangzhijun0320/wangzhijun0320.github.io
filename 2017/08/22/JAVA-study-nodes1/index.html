2.1从hello world 开始

2.1.1 撰写Java原始码

1 定义类
Java程序规定，所有程序代码都要定义在“类”中。class类前有一个public关键词，表示类是公开类，一个Java文档可以定义多个类，但是只能有一个公开类，而且主文档名必须与公开类名相同。
2 定义区块（Block）
程序中XXX类的的区块包括了main()方法，而main()方法的区块包括了xxxx.
3 定义main()方法
Java程序执行的起点是main()方法，mian()方法是public成员，表示可以被JVM公开执行，static表示JVM不用生成类实例就可以调用，Java程序执行过程中的错误都是以外例方式进行处理，所以main()不用传回值，声明为void即可，String【】 args可以在执行程序时，取得用户指定的命令行自变量。
4 撰写描述
println()表示输出字符串后换行，print()表示输出字符串后不换行。

2.1.2PATH是什么

Java的编译工具是Javac，装好JDK后，工具程序放在文件夹bin文件夹中，但是操作系统找不到javac的路径，所以要跟系统说明工具程序放置的位置。
当输入一个指令而没有指定路径信息时，操作系统会依据PATH环境变量中设定的路径顺序，依次寻找各路径下是否有这个指令，可以执行echo %PATH% 来查看当前系统PATH环境变量中包括那些路径信息。

2.1.3JVM(java)与classpath

启动JVM的命令是Java，而要求JVM执行HelloWorld时，只要指定类名，不用附加.class扩展名。
windows的搜索路径是PATH；JVM的搜索路径是CLASSPATH。
JVM预设的classpath就是读取目前文件夹中的class，如果自定义classpath，则以自定义为主。
自定义classpath格式：java -cp c:\xxxxx\xxxx
有时候希望从当前文件夹开始寻找类文档，则可以使用“.”指定。

如果使用了Java链接库，这些链接库中的类文档会封装为JAR文档，也就是扩展名为.jar的文档。jar文档实际使用zip格式压缩，当中包含一堆.class文档。如果使用jar文档，格式为(例)：java -cp c:\xxx\xxx;c:\lib\xxx\xxx\xxx xxx
如果文件夹中有许多.jar文档，从JavaSE6开始，可以使用“*”表示使用文件夹中的所有.jar文档。

2.1.4 编辑程序(javac)与CLASSPATH

当使用javac编译程序时，如果要使用到其他类链接库，也必须指定classpath，告诉Javac编译程序到哪里去找.class。

2.2管理原始码与位码文档

2.2.1编辑程序(javac)与SOURCEPATH

当编译程序时，由于代码中使用其他类，就必须告诉编译程序，其他类的原始码文档放置位置，使用-sourcepath指定文件夹中的原始码文档，而-d指定了编译完成的位码存放文件夹。
实际项目中会有数以万计的类，如果每次都要重新将.java 编译为.class，那么是非常费时间的，因此，应该指定-cp为-d后的文件夹，当文件夹中有需要的.class文件时，无需重新编译。

2.2.3使用import偷懒

JavaSE API中有许多常用类，Java.lang包下的类由于很常用，不用撰写import也可以直接使用class定义的名称。
如果类位于同一包，彼此使用并不需要import。
