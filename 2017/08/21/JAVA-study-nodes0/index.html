Java 平台概括

1.1 Java不只是语言

1.1.1前世今生

1995年5月23日（这一天被公认为Java的诞生日），正是将Oak改名为Java。

Java开发者版本一开始是以 java devalopment kit 名称发表，简称jdk，而J2SE则是平台名称，包含了jdk和Java程序语言。

2006年12月11日发表的Java平台标准版，除了版本号之外，名称也有了变化，称为 java platform standard edition6,简称JavaSE6，JDK6全名则为java se development kit6,也就是不再像以前Java2带有2这个号码，版本号6和1.6.0都使用，6是产品名称，1.6.0是开发者名称。

1.1.2三大平台

Java是各个应用平台的基础，想要学习其他的平台应用，必先了解JavaSE以奠定基础，JavaSE可以分为四个主要的部分：JVM JRE JDK 与Java语言。
为了能够运行Java程，必须有Java虚拟机JVM。JVM包括在JRE中，所以为了要运行Java程序，就必须安装JRE。
如果要开发Java程序，就必须安装JDK，jdk包括jre及开发程序中需要的一些工具程序，像是javac,java,appletviewer等工具程序。

JavaEE是以JavaSE为基础，定义了一系列的服务，api，协议等。适用于开发分布式，多层次，以组件为基础，以web为基础的应用程序，整个JavaEE的体系是相当庞大的。

JavaME是Java平台版中最小的一个，目的是作为小型数字设备上开发及部署应用程序的平台，最熟悉的设备如手机，pda，股票机等。

1.1.3JCP 与JSR

Java不仅是程序语言，还是标准规范。
sun 公司于1998年组成了JCP，这是一个开放性的国际组织。
任何想要提议加入Java的功能或特性，必须以JSR正式文件的方式提交，JSR必须经过JCP执行委员会投票通过，方可成为最终的标准文件。
若JSR成为最终的文件后，必须经过JSR成果做出免费且开发原始码的参考实现，称为RI，并提供TCK做为技术兼容性测试工具箱，方便想要根据JSR实现产品的厂商或组织参与测试兼容性。

1.2JVM JRE JDK

1.2.1什么是JVM

JVM让Java可以跨平台，Java编译时，并不直接编译为相依于某平台的0,1序列，而是翻译为中介格式的位码。Java原始码扩展名为.java ，经过编译程序翻译为扩展名为.class的位码。（编译程序的工具程序由JDK提供）如果想要执行位码，目标平台就必须安装JVM，JVM会将位码翻译为相依于平台的机器码。（位码文档就是JVM的可执行文件）

1.2.2区分JRE和JDK

总结来说就是JDK包括Java程序语言，工具程序和JRE。
JRE则包括了部署技术，java SE API和JVM

1.2.3认识JDK安装内容

JDK本身附带JRE，这个JRE安装在JDK的安装文件夹的jre文件夹中。（Java SE api编译好的.class文件也放在这里面）
bin文件夹中就是Decelopment Tools选项（开发工具就是安装编译程序之类的工具程序）。
src.zip和javafx-src.zip则是Source code选项，也就是Java SE api实现的源代码。
lib目录下面有一个rt.jar文档。是zip格式，使用解压缩软件打开，就会看到好多编译好的.class文档。
