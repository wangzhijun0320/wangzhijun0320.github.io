Chapter4认识对象

类和对象

4.1.1定义类

例子：类名  text=new  类名（）；【有图有真相】

只有有一个类定义，编译程序就是产生一个.class文件。
如果想在建立对象时，一并进行某个初始流程，像是指定数据成员值，则可以定义构造函数。

4.1.2使用标准类

1 使用java.util.Scanner
如果直接取得上一个字符串（以空格或换行分隔），则使用next()；如果想取得用户输入的正行文字，则使用nextLine()（以换行分割）
2 使用java.math.BigDecimal
Java遵从IEEE754浮点数运算规范，使用分数与指数来表示浮点数。由于浮点数误差的关系，就要小心浮点数，不要用==直接比较浮点数的运算结果。
怎么解决这种问题，可以使用java.util.BigDecimal。这个类还提供了plus()，substract(),multiply(),divids()等方法。

4.1.3 对象指定与相等性

Java并非完全的面向对象的程序语言，在Java中有两大类型系统，基本类型和类类型。
如果在操作对象，=是用在指定参考名称参考某个对象，而==是用在比较两个参考名称是否参考用一个对象。通俗的说就是，=是用在将某个名牌绑定到某个对象，而==是用在比较两个名牌是否绑定到了同一个对象。【.equals()可以自行定义如何比较两个对象的内含值】

4.2基本类型打包器

4.2.1打包基本类型

Long、Integer、Double、Float、Boolean等类都是所谓的打包器，正如此名称所示，这些类主要目的就是提供对象实例座位“壳”，将基本类型打包在对象之中，这样就可以操作这个对象。
如果要使用Integer打包int类型数据，方法之一就是使用new创建Integer实例时，传入int类型数据。

4.2.2自动装箱、拆箱

除了使用new关键字创建基本类型打包器外，从j2SE5.0之后提供了自动装箱和自动拆箱功能，可以这样打包基本类型：Integer num=10;//自动装箱 int foo=num;//自动拆箱。

4.2.3 自动装箱，自动拆箱的内幕

自动装箱和自动拆箱实际上是编译程序蜜糖，Integer num=100;编译程序会自动将程序代码展开为：Integer localInteger=Integer.valueOf(100);valueof()的操作数在-128~127中间。所以当Integer i1=200;Integer i2=200;时，i1和i2不会参考同一个实例，所以用==比较就会显示false。用.equals()比较内容是否相等。

4.3数组对象

4.3.1数组基础

数组在Java中就是对象。

4.3.2操作数组对象

int[] scores=new int[10];在Java中看到new，一定是建立对象，这个语法代表了数组就是对象。每个数据类型的数组元素都有初始值，如果默认的初始值不符合你的要求，则可以通过java.util.Arrays的fill()方法来设定数组的元素值。
如果想在new数组中一并指定初始值，则可以不必指定数组长度，int[] scores=new int[]{45,234,24,34,32};
数组既然是对象，对象时根据类而建立的实例，代表建立数组对象定义在哪儿？ 答案是由JVM动态产生。
类类型建立的数组：Integer scores=new Integer[3];那么这个程序片段建立了几个Integer对象呢？是0个。因为如果是类类型，这个片段的写法建立的数组，每个索引都是参考至null。

4.3.3数组复制

如果想进行数组复制，基本做法就是另行建立新数组，使用System.arraycopy()方法，有五个参数，分别是来源数组、来源起始索引、目标数组、目标起始索引、复制长度。还可以使用Arrays.copyOf()方法，这个方法有两个参数，分别是目标对象，目标对象长度。

4.4字符串对象

4.4.1字符串基础

如果有一个char[]数组，也可以使用new来创建String实例，例如：
char[] cs={‘h’,’e’,’l’,’l’,’o’};
String name=new String(cs);
如果有必要，也可以使用String的toCharArray()方法，将字符串以char[]数组返回。
char[] cs2=name.toCharArray();
如果要将字符串转换为整数，浮点数等基本类型，可以使用打包类提供的剖析方法。

4.4.2字符串特性

以“”包含的字符串，只要内容相同，无论在程序代码中出现几次，JVM都只会建立一个String实例，并在字符串池中维护。

4.4.3字符串编码

在windows默认的纯文本编辑器，应该是GB2312编码。在eclipse中建立.java原始码，在eclipse运行在简体中文windows中，默认应该是GB2312编码。然而写下的字符串在JVM中会是Unicode。
