6.1.2多态与is-a

在java中，子类只能继承一个父类，继承除了可以避免类间重复的行为定义外，还有个重要的关系就是子类和父类间还有is-a的关系。
父类不能是一种子类，但是可以强制要求编译程序不要罗嗦。
利用多态思想撰写的函数，函数的参数为父类类型的对象，以抽象方法讲解多态就是使用单一接口操作多种类型的对象。

6.1.3重新定义行为

在继承父类之后，定义与父类中相同的方法部署，但执行内容不同，这称为重新定义(Override)。
在JDK5之后支持标注，其中一个内建的标准标注就是@Override。如果在子类中某个方法前标注@Override，表示要求编译程序检查，该方法是不是真的重新定义了父类中的某个方法，如果不是的话，就会引发编译错误。

6.1.4抽象方法，抽象类

如果某方法区块中没有任何程序代码操作，可以使用 abstract表示该方法为抽象方法，该方法可以不用撰写{}区块，直接用“；”结束即可。
类中如果有抽象方法，表明这个类定义不完整，就不能生成实例，内含 抽象方法的类也必须定义为abstract。
子类如果继承抽象类，有两种方法。一种做法是继续表示该方法为abstract，第二种做法是操作抽象方法。如果都没有实施，那么将编译错误。

##6.2继承语法细节

6.2.1protected 成员

父类中的成员被声明为private，如果子类中的方法想要直接存取父类中的方法，则可以把父类中的方法定义为protected。
被定义为protected的成员。相同包中的类可以直接存取，不同包中的类可以在继承后的子类直接存取。

6.2.2重新定义的细节

如果想取得父类中的方法定义，可以在调用方法前，加上super关键字。
重新定义方法要注意，对于父类中的方法权限，只能扩大但不能缩小。若原来成员为public，子类中重新定义是不可谓private或者protected。

6.2.3再看构造函数

如果类有继承关系，在创建子类实例后，会先进性父类定义的初始流程，在进行子类定义的初始流程。
如果想执行父类中的构造函数，可以使用super()指定。

6.2.4再看final关键字

如果对象数据成员被声明为final，但是没有明确使用=指定值，那表示延迟对象成员的指定，在构造函数执行流程中，一定要有对该数据成员的指定值的动作，否则编译错误。

class前面也可以加上final关键字，如果class前面加上final关键字定义，表示这个类是最后一个了，不会再有子类继承这个类。

定义方法是，也可以限定该方法为final，这表示是最后一次定义方法了，也就是自雷不可以重新定义final方法。

6.2.5java.lang.Object

在Java中，子类只能继承一个父类，如果定义类是没有使用extends关键字指定继承任何类，那么一定是继承java.lang.Object。

如果有个需求是使用数组收集各种对象，那么声明为Object[]。
1 重新定义toString()
许多方法若传入对象，默认都会调用toString()，例如System.out.print()等方法就会调用toString()以取得字符串描述显示。
2 重新定义equals()
instanceof运算符，它可以用来判断对象是否由某个类创建，左操作符是对象，右操作符是类，在使用时，如果左操作符数类型是右操作符数类型的子类型，instanceof也是返回true。

6.2.6关于垃圾回收

创建对象会占用内存，如果程序执行流程中已经无法在使用某个对象，该对象就湖被JVM中的垃圾回收机制回收
